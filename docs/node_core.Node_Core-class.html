Class Node_Core

source code

object --+

        Node_Core

Class which implements a gateway core.

This module has all the function to manage a LoRa connection. We usually use legacy but in mircoPython this functionnality doesn't work porperly. We have an object which gives all LoRa management a frame management for sending frames on a LoRa custom network.

Never use this Module alone, use Gateway

__init__(self, nom="moi", mode='G', nodes={})
(Constructor)

source code 
Parameters:
  • nom (String or bytes) - Name of the node we want to create.
  • mode (Char) - Define the core mode Gateway or not. Put default is set to 'G' to have a gateway core.
  • nodes (Dictionnary) - Allow you to add a list of known node at creation. Avoid to use it.

    Init a Node_Core object. You can init a Gateway core or a classic core. The gateway core is able to manage a list of unknown Node which are trying to communicate with the gateway. When a Node_Core object is created a unique key is created from the id of the device. According to Pycom documentation this id is unique : https://docs.pycom.io/pycom_esp32/library/machine.html#machine.unique_id

_translateIntoKey(self, string, state='receive')

source code 
Parameters:
  • string (16 Bytes.) - Name of the node you want to get the key.
  • state (string) - 'receive' (default) or 'send".

    Give the key corresponding to the node named by string. The name should be known and associated with a key (see AddNode) State indicator is usefull for Gw mode. It indicates if the translation is used for emition or reception. In both case if the node name is unknown the list of Unknown Nodes will updated

Returns:
Key if it exists otherwise it raises KeyError exception.

buildMsg(self, data, dest, encryption=False)

source code 
Parameters:
  • data (String or Bytes.) - Data you want to send.
  • dest (String or Bytes.) - Name of the destinatory node.
  • encryption (Boolean.) - Enable or disable encryption.

    Build a formatted message which can be transmitted with the custom LoRa network.

    See https://github.com/SRimbaud/Passerelle_LoPy/blob/master/scripts/Sch%C3%A9ma_trame.jpg for frame format.

    Note that when you want to crypt your message you need 32 Bytes for transmitting iv.

Returns:
Built message. Can raise KeyError exception if dest is an unknown node (see _translateIntoKey)

readMsg(self, data, encryption=False)

source code 
Parameters:
  • data (Bytes.) - Data you want to read.
  • encryption (Boolean.) - Param to activate decryption methods or not.

    Decrypt a frame built with buildMsg. If the frame was built with encryption mode you have to read it with encryption mode. Note that if the senders of the message is unknown the list of unknown node will be updated and a KeyError will be raised.

Returns:
list of name of senders and message sended.
